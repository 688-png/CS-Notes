## 1. Overview

Regular expressions are used to find and replace text content.

Regular expressions are built into other languages ​​or software products and are not a language or software in themselves.

[Regular expression online tool](https://regexr.com/)

## 2. Match a single character

**.** can be used to match any single character, but in most implementations, it cannot match newlines;

**.** is a metacharacter, meaning it has a special meaning other than the meaning of the character itself. If you need to match . , then use \ to escape, that is, add \ in front of .

Regular expressions are generally case-sensitive, but some implementations are not.

**regular expression**

```
C.C2018
```

**Matching results**

My name is **CyC2018** .

## 3. Match a group of characters

**[ ]** defines a character set;

0-9, a-z define a character interval, the interval is determined by ASCII code, and the character interval is used in [ ].

**-** Only between [ ] is a metacharacter, and outside [ ] is an ordinary character;

**^** is a negation operation in [ ].

**Application**

Matches strings that begin with abc and whose last letter is not a number:

**regular expression**

```
abc[^0-9]
```

**Matching results**

1. **abcd**
2.abc1
3.abc2

## 4. Use metacharacters

### Match whitespace characters

| metacharacter | description |
| :---: | :---: |
| [\b] | Go back (delete) one character |
| \f | form feed character |
| \n | newline character |
| \r | carriage return |
| \t | tab character |
| \v | vertical tab character |

\r\n is the text line ending tag in Windows and \n in Unix/Linux.

\r\n\r\n can match blank lines under Windows because it matches two consecutive end-of-line tags, which are the blank lines between two records;

### Match specific characters

#### 1. Numeric metacharacters

| metacharacter | description |
| :---: | :---: |
| \d | Numeric character, equivalent to [0-9] |
| \D | Non-numeric characters, equivalent to [^0-9] |

#### 2. Alphanumeric metacharacters

| metacharacter | description |
| :---: | :---: |
| \w | Uppercase and lowercase letters, underscores and numbers, equivalent to [a-zA-Z0-9\_] |
| \W | Negate \w |

#### 3. White space metacharacter

| metacharacter | description |
| :---: | :---: |
| \s | Any whitespace character, equivalent to [
\f\n\r\t\v] |
| \S | Negate \s |

\x matches hexadecimal characters, \0 matches octal, for example \xA corresponds to the ASCII character with value 10, which is \n.

## 5. Repeat matching

- **\+** matches 1 or more characters
- **\** * matches 0 or more characters
- **?** matches 0 or 1 characters

**Application**

Match email addresses.

**regular expression**

```
[\w.]+@\w+\.\w+
```

[\w.] matches letters or numbers. Add + after it to match multiple times. In the character set [ ], . is not a metacharacter;

**Matching results**

**abc.def\<span\>@\</span\>qq.com**

- **{n}** matches n characters
- **{m,n}** matches m\~n characters
- **{m,}** matches at least m characters

\* and + are both greedy metacharacters and will match as much content as possible. Adding a trailing ? converts to lazy metacharacters such as \*?, +?, and {m,n}?.

**regular expression**

```
a.+c
```

**Matching results**

**abcabcabc**

Since + is greedy, .+ will match more content, so it will match the entire abcabcabc text instead of just the preceding abc text. Lazy type can be used to match the previous one.

## 6. Position matching

### Word boundaries

**\b** can match a word boundary, which is a position between \w and \W; **\B** matches a position that is not a word boundary.

\b only matches positions, not characters, so \babc\b matches 3 characters.

### String boundaries

**^** matches the beginning of the entire string, **$** matches the end.

The ^ metacharacter is used to negate within a character set and to match the beginning of a string outside of a character set.

In line-break matching mode (multiline), newlines are treated as string boundaries.

**Application**

Matches comment lines starting with // in code

**regular expression**

```
^\s*\/\/.*$
```

## 7. Use subexpressions

Use **( )** to define a subexpression. The content of a subexpression can be treated as an independent element, that is, it can be treated as a character, and metacharacters such as * can be used.

Subexpressions can be nested, but if the nesting level is too deep, it will become difficult to understand.

**regular expression**

```
(ab){2,}
```

**Matching results**

**ababab**

**|**
Is the or metacharacter, which treats all parts on the left and right as two separate parts, as long as one of the two parts matches.

**regular expression**

```
(19|20)\d{2}
```

**Matching results**

1. **1900**
2. **2010**
3. 1020

**Application**

Match IP address.

Each part of the IP address is a number from 0-255. When matching with regular expressions, the following situations are legal:

-
one digit
- Two digits not starting with 0
- three digits starting with 1
- Starting with 2, the second digit is a three-digit number from 0 to 4
- Starting with 25, the third digit is a three-digit number from 0 to 5

**regular expression**

```
((25[0-5]|(2[0-4]\d)|(1\d{2})|([1-9]\d)|(\d))\.){3}(25[0-5]|(2[0-4]\d)|(1\d{2})|([1-9]\d)|(\d))
```

**Matching results**

1. **192.168.0.1**
2. 00.00.00.00
3. 555.555.555.555

## 8. Back reference

Backreference uses **\n** to refer to a subexpression, where n represents the sequence number of the subexpression, starting from 1. It is consistent with the content matched by the subexpression. For example, if the subexpression matches abc, then the backreference part also needs to match abc.

**Application**

Matches valid title elements in HTML.

**regular expression**

\1 will refer back to the content matched by the subexpression (h[1-6]), which means it must be consistent with the content matched by the subexpression.

```
<(h[1-6])>\w*?<\/\1>
```

**Matching results**

1. **&lt;h1\>x&lt;/h1\>**
2. **&lt;h2\>x&lt;/h2\>**
3. &lt;h3\>x&lt;/h1\>

### Replacement

Two regular expressions are required.

**Application**

Modify the phone number format.

**text**

313-555-1234

**Find regular expression**

```
(\d{3})(-)(\d{3})(-)(\d{4})
```

**Replace regular expression**

Add () to the result of the first subexpression search, then add a space, and add - to separate the results of the third and fifth subexpression searches.

```
($1) $3-$5
```

**Results**

(313) 555-1234

### Case conversion

| metacharacter | description |
| :---: | :---: |
| \l | Convert next character to lowercase |
| \u| Convert next character to uppercase |
| \L | Convert all characters between \L and \E to lowercase |
| \U | Convert all characters between \U and \E to uppercase |
| \E | end \L or \U |

**Application**

Converts the second and third characters of text to uppercase.

**text**

abcd

**Find**

```
(\w)(\w{2})(\w)
```

**Replace**

```
$1\U$2\E$3
```

**Results**

ikB

## 9. Search before and after

The front and back search stipulates the content that should be matched at the beginning and end of the matched content, but does not include the matching content at the beginning and end.

Forward search is defined using **?=**, which specifies the tail matching content, which is defined after ?=. The so-called forward search is to specify a matching content, and then use this content as the tail to search for the content that needs to be matched. Backward matching is defined with ?\<= (Note: JavaScript does not support backward matching, and Java's support for it is not complete).

**Application**

Find the part of the email address before the @ character.

**regular expression**

```
\w+(?=@)
```

**Results**

**abc** @qq.com

To negate forward and backward searches, just replace = with !, for example, replace (?=) with (?!). The negation operation matches content that does not meet the requirements at the beginning and e
nd.

## 10. Embedding conditions

### Backtracking reference conditions

The condition is whether a certain subexpression matches. If it matches, it needs to continue to match the content after the conditional expression.

**regular expression**

The subexpression (\\() matches a left bracket, and the following ? means matching 0 or 1. ?(1) is the condition. When the subexpression 1 matches, the condition is established, and \) matching needs to be performed, that is, matching the right bracket.

```
(\()?abc(?(1)\))
```

**Results**

1. **(abc)**
2. **abc**
3. (abc

### Search conditions before and after

The condition is whether the beginning and end of the definition match. If they match, continue to perform subsequent matching. Note that the first and last characters are not included in the matched content.

**regular expression**

 ?(?=-) is the forward search condition. Only if \d{5} can be matched at the end of the forward search with -, then -\d{4} will continue to be matched.

```
\d{5}(?(?=-)-\d{4})
```

**Results**

1. **11111**
2. 22222-
3. **33333-4444**
