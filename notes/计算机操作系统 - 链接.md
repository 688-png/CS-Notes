## Compilation system


The following is a hello.c program:

```c
#include <stdio.h>

int main()
{
    printf("hello, world\n");
    return 0;
}
```

On Unix systems, the compiler converts source files into object files.

```bash
gcc -o hello hello.c
```

The process is roughly as follows:

- Preprocessing stage: processing preprocessing commands starting with #;
- Compilation stage: translated into assembly files;
- Assembly stage: Translate assembly files into relocatable object files;
- Linking stage: Merge relocatable target files and separately precompiled target files such as printf.o to obtain the final executable target file.

## Static linking

The static linker takes a set of relocatable object files as input and produces a fully linked executable object file as output. The linker mainly completes the following two tasks:

- Symbol resolution: Each symbol corresponds to a function, a global variable or a static variable. The purpose of symbol resolution is to associate each symbol reference with a symbol definition.
- Relocation: The linker associates each symbol definition with a memory location and then modifies all references to these symbols so that they point to this memory location.

## Target file

- Executable object file: can be executed directly in memory;
- Relocatable object file: can be merged with other relocatable object files during the link stage to create an executable object file;
- Shared object file: This is a special relocatable object file that can be dynamically loaded into memory and linked at runtime;

## Dynamic link

Static libraries have the following two problems:

- When the static library is updated, the entire program must be relinked;
- For standard function libraries like printf, if every program needs to have code, this will be a huge waste of resources.

Shared libraries are designed to solve these two problems of static libraries. They are usually represented by the .so suffix on Linux systems and they are called DLLs on Windows systems. It has the following characteristics:

- There is only one file for a library in a given file system. This file is shared by all executable object files that reference the library. It will not be copied to the executable files that reference it;
- In memory, a copy of a shared library's .text section (the machine code of the compiled program) can be shared by different running pro
cesses.
