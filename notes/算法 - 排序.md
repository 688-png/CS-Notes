# Algorithm - Sorting
## Agreement

The elements to be sorted need to implement Java's Comparable interface, which has a compareTo() method, which can be used to determine the size relationship between two elements.

Use the auxiliary functions less() and swap() to perform comparison and exchange operations, making the code more readable and portable.

The cost model of a sorting algorithm is the number of comparisons and exchanges.

```java
public abstract class Sort<T extends Comparable<T>> {

    public abstract void sort(T[] nums);

    protected boolean less(T v, T w) {
        return v.compareTo(w) < 0;
    }

    protected void swap(T[] a, int i, int j) {
        T t = a[i];
        a[i] = a[j];
        a[j] = t;
    }
}
```

## Select sort

Selects the smallest element from the array and swaps it with the first element of the array. Then select the smallest element from the remaining elements in the array and swap it with the second element of the array. Keep doing this until the entire array is sorted.

Selection sort requires \~N<sup>2</sup>/2 comparisons and \~N exchanges. Its running time has nothing to do with the input. This feature makes it require so many comparisons and exchanges for a sorted array.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/bc6be2d0-ed5e-4def-89e5-3ada9afa811a.gif" width="230px"> </div><br>

```java
public class Selection<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {
        int N = nums.length;
        for (int i = 0; i < N - 1; i++) {
            int min = i;
            for (int j = i + 1; j < N; j++) {
                if (less(nums[j], nums[min])) {
                    min = j;
                }
            }
            swap(nums, i, min);
        }
    }
}
```

## Bubble sort

Continuously exchange adjacent elements in reverse order from left to right. After a round of looping, the largest unsorted element can be floated to the right.

In a loop, if no exchange occurs, it means that the array is already in order, and you can exit directly at this time.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/0f8d178b-52d8-491b-9dfd-41e05a952578.gif" width="200px"> </div><br>

```java
public class Bubble<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {
        int N = nums.length;
        boolean isSorted = false;
        for (int i = N - 1; i > 0 && !isSorted; i--) {
            isSorted = true;
            for (int j = 0; j < i; j++) {
                if (less(nums[j + 1], nums[j])) {
                    isSorted = false;
                    swap(nums, j, j + 1);
                }
            }
        }
    }
}
```

## Insertion sort

Each time, the current element is inserted into the sorted array on the left, so that the array on the left is still in order after insertion.

For the array {3, 5, 2, 4, 1}, it has the following reverse order
: (3, 2), (3, 1), (5, 2), (5, 4), (5, 1), (2, 1), (4, 1). Insertion sort can only exchange adjacent elements each time, reducing the number of reverse orders by 1, so the number of exchanges required for insertion sort is the number of reverse orders.

The time complexity of insertion sort depends on the initial order of the array. If the array is already partially ordered, then there are fewer reverse orders, fewer exchanges are required, and the time complexity is lower.

- On average, insertion sort requires \~N<sup>2</sup>/4 comparisons and \~N<sup>2</sup>/4 exchanges;
- In the worst case, \~N<sup>2</sup>/2 comparisons and \~N<sup>2</sup>/2 exchanges are required. In the worst case, the array is in reverse order;
- In the best case, N-1 comparisons and 0 swaps are required. In the best case, the array is already sorted.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/35253fa4-f60a-4e3b-aaec-8fc835aabdac.gif" width="200px"> </div><br>

```java
public class Insertion<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {
        int N = nums.length;
        for (int i = 1; i < N; i++) {
            for (int j = i; j > 0 && less(nums[j], nums[j - 1]); j--) {
                swap(nums, j, j - 1);
            }
        }
    }
}
```

## Hill sorting

For large arrays, insertion sort is slow because it can only swap adjacent elements, reducing the reverse order by one at a time. The emergence of Hill sorting is to solve this limitation of insertion sort. It can reduce the number of reverse orders by more than 1 each time by exchanging non-adjacent elements.

Hill sort uses insertion sort to pair gaps
The sequence of h is sorted. By continuously reducing h, and finally setting h=1, the entire array can be made ordered.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/7818c574-97a8-48db-8e62-8bfb030b02ba.png" width="450px"> </div><br>

```java
public class Shell<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {

        int N = nums.length;
        int h = 1;

        while (h < N / 3) {
            h = 3 * h + 1; // 1, 4, 13, 40, ...
        }

        while (h >= 1) {
            for (int i = h; i < N; i++) {
                for (int j = i; j >= h && less(nums[j], nums[j - h]); j -= h) {
                    swap(nums, j, j - h);
                }
            }
            h = h / 3;
        }
    }
}

```

The running time of Hill sort cannot reach the square level. The number of comparisons required for Hill sort using the increasing sequence 1, 4, 13, 40, ... will not exceed several times of N multiplied by the length of the increasing sequence. The advanced sorting algorithm introduced later is only about two times faster than Hill sorting.

## Merge sort

The idea of merge sort is to divide the array into two parts, sort them separately, and then merge them together.

<div align="center"
> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/ec840967-d127-4da3-b6bb-186996c56746.png" width="300px"> </div><br>

### 1. Merge method

The merge method merges two sorted parts of an array into one.

```java
public abstract class MergeSort<T extends Comparable<T>> extends Sort<T> {

    protected T[] aux;


    protected void merge(T[] nums, int l, int m, int h) {

        int i = l, j = m + 1;

        for (int k = l; k <= h; k++) {
            aux[k] = nums[k]; // Copy data to auxiliary array
        }

        for (int k = l; k <= h; k++) {
            if (i > m) {
                nums[k] = aux[j++];

            } else if (j > h) {
                nums[k] = aux[i++];

            } else if (aux[i].compareTo(aux[j]) <= 0) {
                nums[k] = aux[i++]; // Do this step first to ensure stability

            } else {
                nums[k] = aux[j++];
            }
        }
    }
}
```

### 2. Top-down merge sorting

Divide a large array into two small arrays to solve.

Because the problem is divided in half into two sub-problems each time, the complexity of this half-divided algorithm is generally O(NlogN).

```java
public class Up2DownMergeSort<T extends Comparable<T>> extends MergeSort<T> {

    @Override
    public void sort(T[] nums) {
        aux = (T[]) new Comparable[nums.length];
        sort(nums, 0, nums.length - 1);
    }

    private void sort(T[] nums, int l, int h) {
        if (h <= l) {
            return;
        }
        int mid = l + (h - l) / 2;
        sort(nums, l, mid);
        sort(nums, mid + 1, h);
        merge(nums, l, mid, h);
    }
}
```


### 3. Bottom-up merge sorting

Merge those mini-arrays first, then merge the resulting mini-arrays in pairs.

```java
public class Down2UpMergeSort<T extends Comparable<T>> extends MergeSort<T> {

    @Override
    public void sort(T[] nums) {

        int N = nums.length;
        aux = (T[]) new Comparable[N];

        for (int sz = 1; sz < N; sz += sz) {
            for (int lo = 0; lo < N - sz; lo += sz + sz) {
                merge(nums, lo, lo + sz - 1, Math.min(lo + sz + sz - 1, N - 1));
            }
        }
    }
}

```

## Quick sort

### 1. Basic algorithm

- Merge sort divides the array into two sub-arrays and sorts them separately, and merges the ordered sub-arrays so that the entire array is sorted;
- Quick sort divides the array into two sub-arrays through a split element. The left sub-array is less than or equal to the split element, and the right sub-array is greater than or equal to the split element. Sorting these two sub-arrays will sort the entire array.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/6234eb3d-ccf2-4987-a72

```java
public class QuickSort<T extends Comparable<T>> extends Sort<T> {

    @Override
    public void sort(T[] nums) {
        shuffle(nums);
        sort(nums, 0, nums.length - 1);
    }

    private void sort(T[
] nums, int l, int h) {
        if (h <= l)
            return;
        int j = partition(nums, l, h);
        sort(nums, l, j - 1);
        sort(nums, j + 1, h);
    }

    private void shuffle(T[] nums) {
        List<Comparable> list = Arrays.asList(nums);
        Collections.shuffle(list);
        list.toArray(nums);
    }
}
```

### 2. Segmentation

Take a[l] as the split element, then scan from the left end of the array to the right until you find the first element that is greater than or equal to it, then scan from the right end of the array to the left to find the first element that is less than it, and exchange the two elements. By continuing this process, you can ensure that the left element of the left pointer i is not larger than the segmented element, and the right element of the right pointer j is not smaller than the segmented element. When the two pointers meet, the slice elements a[l] and a[j] are swapped.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/c4859290-e27d-4f12-becf-e2a5c1f3a275.gif" width="320px"> </div><br>

```java
private int partition(T[] nums, int l, int h) {
    int i = l, j = h + 1;
    T v = nums[l];
    while (true) {
        while (less(nums[++i], v) && i != h) ;
        while (less(v, nums[--j]) && j != l) ;
        if (i >= j)
            break;
        swap(nums, i, j);
    }
    swap(nums, l, j);
    return j;
}
```

### 3. Performance analysis

Quick sort is an in-place sort and does not require an auxiliary array, but recursive calls require an auxiliary stack.

The best case scenario for quick sort is to divide the array in half exactly every time, so that the number of recursive calls is minimized. In this case, the number of comparisons is C<sub>N</sub>=2C<sub>N/2</sub>+N, and the complexity is O(NlogN).

In the worst case, the first split is from the smallest element, the second split is from the second smallest element, and so on. So in the worst case N<sup>2</sup>/2 needs to be compared. In order to prevent the array from being ordered at the beginning, the array needs to be randomly shuffled when performing quick sort.

### 4. Algorithm improvements

##### 4.1 Switch to insertion sort

Because quick sort will also call itself recursively in small arrays, insertion sort has better performance than quick sort for small arrays, so you can switch to insertion sort in small arrays.

##### 4.2 Hit the right number among three numbers

In the best case, the median of the array can be taken as the split element every time, but calculating the median is very expensive. A compromise is to take 3 elements and use the middle-sized element as the split element.

##### 4.3 Three-way segmentation

For arrays with a large number of repeated elements, the array can be divided into three parts, corresponding to the less than, equal to, and greater than elements.

Three-way split quick sort can sort random arrays with a large number of repeated elements in linear time.

``
`java
public class ThreeWayQuickSort<T extends Comparable<T>> extends QuickSort<T> {

    @Override
    protected void sort(T[] nums, int l, int h) {
        if (h <= l) {
            return;
        }
        int lt = l, i = l + 1, gt = h;
        T v = nums[l];
        while (i <= gt) {
            int cmp = nums[i].compareTo(v);
            if (cmp < 0) {
                swap(nums, lt++, i++);
            } else if (cmp > 0) {
                swap(nums, i, gt--);
            } else {
                i++;
            }
        }
        sort(nums, l, lt - 1);
        sort(nums, gt + 1, h);
    }
}
```

### 5. Fast selection algorithm based on segmentation

The partition() method of quick sort will return an integer j such that a[l..j-1] is less than or equal to a[j], and a[j+1..h] is greater than or equal to a[j]. In this case, a[j] is the j-th largest element of the array.

You can use this property to find the kth element of an array.

The algorithm is linear-level. Assuming that the array can be divided into two parts each time, the total number of comparisons is (N+N/2+N/4+..) until the k-th element is found. This sum is obviously less than 2N.

```java
public T select(T[] nums, int k) {
    int l = 0, h = nums.length - 1;
    while (h > l) {
        int j = partition(nums, l, h);

        if (j == k) {
            return nums[k];

        } else if (j > k) {
            h = j - 1;

        } else {
            l = j + 1;
        }
    }
    return nums[k];
}
```

## Heap sort

### 1. Heap

The value of a node in the heap is always greater than or equal to or less than or equal to the value of its child node, and the heap is a complete binary tree.

The heap can be represented by an array because the heap
is a complete binary tree, and a complete binary tree can easily be stored in an array. The parent node of the node at position k has position k/2, and its two child nodes have positions 2k and 2k+1 respectively. The position with array index 0 is not used here in order to describe the positional relationship of nodes more clearly.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/f48883c8-9d8a-494e-99a4-317d8ddb8552.png" width="170px"> </div><br>

```java
public class Heap<T extends Comparable<T>> {

    private T[] heap;
    private int N = 0;

    public Heap(int maxN) {
        this.heap = (T[]) new Comparable[maxN + 1];
    }

    public boolean isEmpty() {
        return N == 0;
    }

    public int size() {
        return N;
    }

    private boolean less(int i, int j) {
        return heap[i].compareTo(heap[j]) < 0;
    }

    private void swap(int i, int j) {
        T t = heap[i];
        heap[i] = heap[j];
        heap[j] = t;
    }
}
```

### 2. Floating and sinking

In the heap, when a node is larger than its parent node, the two nodes need to be swapped. After the exchange, it may be larger than its new parent node, so constant comparison and exchange operations are required. This ope
ration is called floating.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/99d5e84e-fc2a-49a3-8259-8de274617756.gif" width="270px"> </div><br>

```java
private void swim(int k) {
    while (k > 1 && less(k / 2, k)) {
        swap(k / 2, k);
        k = k / 2;
    }
}
```

Similarly, when a node is smaller than its child nodes, it also needs to continuously perform downward comparison and exchange operations. This operation is called sinking. If a node has two child nodes, it should be exchanged with the largest of the two child nodes.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/4bf5e3fb-a285-4138-b3b6-780956eb1df1.gif" width="270px"> </div><br>

```java
private void sink(int k) {
    while (2 * k <= N) {
        int j = 2 * k;
        if (j < N && less(j, j + 1))
            j++;
        if (!less(k, j))
            break;
        swap(k, j);
        k = j;
    }
}
```

### 3. Insert elements

Place the new element at the end of the array and float it to the appropriate position.

```java
public void insert(Comparable v) {
    heap[++N] = v;
    swim(N);
}
```

### 4. Delete the largest element

Remove the largest element from the top of the array, place the last element of the array at the top, and let this element sink to the appropriate position.

```java
public T delMax() {
    T max = heap[1];
    swap(1, N--);
    heap[N + 1] = null;
    sink(1);
    return max;
}
```

### 5. Heap sort

By exchanging the position of the largest element with the last element of the array in the current heap without deleting it, you can get a decreasing sequence from the end to the beginning, which is an increasing sequence from the forward perspective. This is heap sorting.

##### 5.1 Build heap

The most direct way to create a heap from an unordered array is to traverse the array from left to right and perform floating operations. A more efficient method is to perform a sinking operation from right to left. If both nodes of a node are already in heap order, then performing a sinking operation can make the heap of this node the root node in order. Leaf nodes do not need to be sinked, and the elements of leaf nodes can be ignored, so only half of the elements need to be traversed.

##### 5.2 Exchange the top element and the last element of the heap

After the exchange, a sinking operation is required to maintain the orderly state of the heap.

        return nums[i].compareTo(nums[j]) < 0;

### 6. Analysis

The height of a heap is logN, so the complexity of inserting elements and deleting the largest element in the heap is logN.

For heap sort, the complexity is NlogN due to the sinking operation of N nodes.

Heap sort is an in-place sort that does not utilize additional space.

Modern operating systems rarely use heap sorting because it cannot take advantage of the locality principle for caching, that is, array elements are rarely compared and exchanged with adjacent elements.

## Summary

### 1. Comparison of sorting algorithms

| Algorithm | Stability | Time complexity | Space complexity | Remarks |
| :---: | :---: |:---: | :---: | :---: |
| Selection sort | × | N<sup>2</sup> | 1 | |
| Bubble sort | √ | N<sup>2</sup> | 1 | |
| Insertion sort | √ | N \~ N<sup>2</sup> | 1 | Time complexity is related to the initial order |
| Hill sort | × | Several times of N multiplied by the length of the increasing sequence | 1 | Improved insertion sort |
| Quick sort | × | NlogN | logN | |
| Three-way split quick sort | × | N \~ NlogN | logN | Suitable for a large number of duplicate primary keys |
| Merge sort | √ | NlogN | N | |
| Heap sort | × | NlogN | 1 | Unable to take advantage of locality principle |

Quick sort is the fastest general-purpose sorting algorithm. Its inner loop requires few instructions, and it can also take advantage of cache because it always accesses data sequentially. Its running time is approximately \~cNlogN, where c is smaller than other linear logarithmic sorting algorithms.

Using three-way split quick sort, certain distributions of inputs that may occur in practical applications can achieve linear levels, while other sorting algorithms still require linear logarithmic time.

### 2. Java sorting algorithm implementation

The main sorting method in Java is java.util.Arrays.sort(), which uses three-way splitting quick sort for primitive data types and merge sort for reference types.
