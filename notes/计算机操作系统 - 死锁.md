## Necessary conditions

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/c037c901-7eae-4e31-a1e4-9d41329e5c3e.png"/> </div><br>

- Mutually exclusive: Each resource is either already allocated to a process or is available.
- Possession and waiting: A process that has already obtained a resource can request a new resource.
- Non-preemptible: A resource that has been allocated to a process cannot be forcibly preempted. It can only be explicitly released by the process that owns it.
- Loop waiting: Two or more processes form a loop, and each process in the loop is waiting for the resources occupied by the next process.

## Processing method

There are mainly four methods:

- Ostrich strategy
- Deadlock detection and deadlock recovery
- Deadlock prevention
- Deadlock avoidance

## Ostrich strategy

Bury your head in the sand and pretend nothing is wrong.

Because solving the deadlock problem is very expensive
High, so the ostrich strategy, which does not take task measures, will achieve higher performance.

When a deadlock does not have much impact on users, or the probability of a deadlock is very low, the ostrich strategy can be used.

Most operating systems, including Unix, Linux, and Windows, deal with the deadlock problem simply by ignoring it.

## Deadlock detection and deadlock recovery

Does not attempt to prevent deadlocks, but instead takes steps to recover when a deadlock is detected.

### 1. Deadlock detection for one resource per type

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/b1fa0453-a4b0-4eae-a352-48acca8fff74.png"/> </div><br>

The above figure is a resource allocation diagram, in which the boxes represent resources and the circles represent processes. The resource pointing to the process means that the resource has been allocated to the process, and the process pointing to the resource means that the process requests to obtain the resource.

Figure a can extract the loop, as shown in figure b, which satisfies the loop waiting condition, so deadlock will occur.

The deadlock detection algorithm for each type of resource is implemented by detecting whether there is a cycle in the directed graph. It starts from a node and performs a depth-first search to mark the visited nodes. If the marked node is visited, it means that there is a cycle in the directed graph, that is, the occurrence of deadlock is detected.

### 2. Deadlock detection for multiple resources of each type

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/e1eda3d5-5ec8-4708-8e25-1a04c5e11f48.png"/> </div><br>

In the picture above, there are three processes and four resources. The meaning of each data is as follows:

- E vector: total resources
- A vector: remaining amount of resources
- C matrix: the number of resources owned by each process, each row represents the number of resources owned by a process
- R matrix: number of resources requested by each process

The resources requested by processes P<sub>1</sub> and P<sub>2</sub> cannot be satisfied. Only process P<sub>3</sub> can let P<sub>3</sub> execute and then release the resources owned by P<sub>3</sub>. At this time, A = (2 2 2 0). P<sub>2</sub> can be executed, and the resources owned by P<sub>2</sub> are released after execution, A = (4 2 2 1). P
<sub>1</sub> can also be executed. All processes execute smoothly without deadlocks.

The algorithm is summarized as follows:

Each process is not marked initially, and the execution process may be marked. When the algorithm ends, any process that is not marked is a deadlocked process.

1. Find an unmarked process P<sub>i</sub> whose requested resource is less than or equal to A.
2. If such a process is found, add the i-th row vector of the C matrix to A, mark the process, and switch back to 1.
3. If there is no such process, the algorithm terminates.

### 3. Banker algorithm for multiple resources

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/62e0dd4f-44c3-43ee-bb6e-fedb9e068519.png"/> </div><br>

There are five processes and four resources in the picture above. The picture on the left represents the resources that have been allocated, and the picture on the right represents the resources that still need to be allocated. The rightmost E, P, and A represent: total resources, allocated resources, and available resources respectively. Note that these three are vectors, not specific values. For example, A=(1020), which means that 1/0/2/0 of the 4 resources are left respectively.

The algorithm for checking whether a state is safe is as follows:

- Find if there is a row in the matrix on the right that is less than or equal to vector A. If no such row exists, the system will deadlock and the state is unsafe.
- If such a line is found, mark the process as terminated and add its allocated resources to A.
- Repeat the above two steps until all processes are marked as terminated and the state is safe.

If a state is not safe, entry into the state needs to be denied.
