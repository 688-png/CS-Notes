## Overview

Because the network layer is the core of the entire Internet, it should be kept as simple as possible. The network layer upwards only provides simple and flexible, connectionless, best-effort interactive datagram services.

Using the IP protocol, heterogeneous physical networks can be connected so that the network layer looks like a unified network.

There are three protocols used in conjunction with the IP protocol:

- Address Resolution Protocol ARP (Address Resolution Protocol)
- Internet Control Message Protocol ICMP (Internet Control Message Protocol)
- Internet Group Management Protocol IGMP (Internet Group Management Protocol)

## IP datagram format

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/85c05fb1-5546-4c50-9221-21f231cdc8c5.jpg" width="700"/> </div><br>

- **Version**: There are two values ​​4 (IPv4) and 6 (IPv6);

- **Header Length**: 4 bits, so the maximum value is 15. A value of 1 represents the length of a 32-bit word, which is 4 bytes. Because the fixed part length is 20 bytes, the minimum value is 5. If the length of the optional field is not an integer multiple of 4 bytes, it is padded with trailing padding.

- **Differentiated Services**: Used to obtain better services, generally not used.

- **Total length**: Including header length and data part length.

- **Time to Live**: TTL, which exists to prevent undeliverable datagrams from circulating around the Internet. Measured in router hops, datagrams are dropped when the TTL is 0.

- **Protocol**: Indicate which protocol the carried data should be handed over for processing, such as ICMP, TC
P, UDP, etc.

- **Header Checksum**: Because the checksum must be recalculated every time the datagram passes through a router, the checksum does not include the data part to reduce the calculation workload.

- **Identifier**: In the case where datagram length is too long and fragmentation occurs, different fragments of the same datagram have the same identifier.

- **Shard Offset**: Together with the identifier, used when fragmentation occurs. The unit of slice offset is 8 bytes.

## IP address addressing method

The addressing method of IP address has gone through three historical stages:

- Classification
- Subnetting
- No classification

### 1. Classification

It consists of two parts, the network number and the host number. Different categories have different network number lengths, and they are fixed.

IP address::= {\<network number\>, \<host number\>}

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/cbf50eb8-22b4-4528-a2e7-d187143d57f7.png" width="500"/> </div><br>

### 2. Subnetting

By taking part of the host number field as the subnet number, the two-level IP address is divided into a third-level IP address.

IP address ::= {\<network number\>, \<subnet number\>, \<host number\>}

To use subnets, a subnet mask must be configured. The default subnet mask of a Class B address is 255.255.0.0. If the subnet of a Class B address occupies two bits, the subnet mask is 11111111 11111111 11000000 00000000, which is 255.255.192.0.

Note that the existence of the subnet is not visible to external networks.

### 3. No classification

Classless addressing CIDR eliminates the traditional concepts of Class A, Class B, and Class C addresses and subnetting, and uses a network prefix and host number to encode the IP address. The length of the network prefix can be changed as needed.

IP address ::= {\<network prefix number\>, \<host number\>}

CIDR is notated by adding the network prefix length after the IP address. For example, 128.14.35.7/20 means that the first 20 digits are the network prefix.

The CIDR address mask can continue to be called a subnet mask, and the length of the first 1 of the subnet mask is the length of the network prefix.

There are many addresses in a CIDR address block, and the network represented by one CIDR can represent many original networks, and only one route is needed in the routing table to replace the original multiple routes, reducing the number of routing table entries. This method of reducing routing table entries by using network prefixes is called route aggregation, also known as forming a supernet.

The items in the routing table are composed of "network prefix" and "next hop address". When searching, you may get more than one matching result. The longest prefix matching should be used to determine which one should be m
atched.

## Address Resolution Protocol ARP

The network layer implements communication between hosts, while the link layer implements communication between specific links. Therefore, during the communication process, the source address and destination address of the IP datagram always remain unchanged, while the MAC
The address changes as the link changes.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/66192382-558b-4b05-a35d-ac4a2b1a9811.jpg" width="700"/> </div><br>

ARP implements obtaining the MAC address from the IP address.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/b9d79a5a-e7af-499b-b989-f10483e71b8b.jpg" width="500"/> </div><br>

Each host has an ARP cache, which contains a mapping table from the IP address to the MAC address of each host and router on the local area network.

If host A knows the IP address of host B, but there is no mapping from the IP address to the MAC address in the ARP cache, host A sends an ARP request packet by broadcast. After receiving the request, host B will send an ARP response packet to host A to inform it of its MAC address. Then host A writes the mapping of host B's IP address to MAC address into its cache.

## Internet Control Message Protocol ICMP

ICMP is designed to forward IP datagrams more efficiently and improve the chances of successful delivery. It is encapsulated in IP datagrams but is not a higher-level protocol.

ICMP messages are divided into error report messages and query messages.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/aa29cc88-7256-4399-8c7f-3cf4a6489559.png" width="600"/> </div><br>

### 1. Ping

Ping is an important application of ICMP, mainly used to test the connectivity between two hosts.

The principle of Ping is to send an ICMP Echo request message to the destination host, and the destination host will send an Echo reply message after receiving it. Ping estimates packet round-trip time and packet loss rate based on time and the number of successful responses.

### 2. Traceroute

Traceroute is another application of ICMP that is used to trace the path of a packet from source to destination.

The IP datagram sent by Traceroute encapsulates the UDP user datagram that cannot be delivered, and the destination host sends a destination unreachable error report message.

- The source host sends a series of IP datagrams to the destination host. The TTL of the first datagram P1 is set to 1. When P1 reaches the first router R1 on the path, R1 accepts it and decrements the TTL by 1. At this time, the TTL is equal to 0, and R1 discards P1 and sends
an ICMP time exceeded error report message to the source host;
- The source host then sends a second datagram, P2, with the TTL set to 2. P2 arrives at R1 first. After receiving the packet, R1 decrements the TTL by 1 and then forwards it to R2. After receiving the packet, R2 also decrements the TTL by 1. Since the TTL is equal to 0 at this time, R2 discards P2 and sends an ICMP time exceeded error message to the source host.
- Continue to perform such steps until the last datagram has just arrived at the destination host, and the host does not forward the datagram or decrement the TTL value by 1. However, because the datagram is encapsulated in UDP that cannot be delivered, the destination host needs to send an ICMP endpoint unreachable error report message to the source host.
- The source host then knows the IP addresses of the routers it passes to reach the destination host and the round-trip time to each router.

## Virtual Private Network VPN

Due to the shortage of IP addresses, the number of IP addresses an organization can apply for is often far less than the number of hosts owned by the organization. And an organization does not need to connect all hosts to the external Internet. Computers within the organization can use IP addresses (private addresses) that are only valid within the organization.

There are three dedicated address blocks:

- 10.0.0.0 \~ 10.255.255.255
- 172.16.0.0 \~ 172.31.255.255
- 192.168.0.0 \~ 192.168.255.255

VPN uses the public Internet as the communication carrier between the organization's private networks. Private means that the host within the organization only communicates with other hosts in the organization; virtual means that it appears to be, but in fact is not, it has access to the public Internet.

The datagram is first sent to router R1, which is connected to the Internet. R1 encrypts the internal data and then re-adds the header of the datagram. The source address is the global address of router R1, 125.1.2.3, and the destination address is the global address of router R2, 194.4.5.6. After receiving the datagram, router R2 decrypts the data part and restores the original datagram. At this time, the destination address is 10.2.0.3, and it is delivered to Y.

## Network Address Translation NAT

When a host inside a private network uses a local IP address and wants to communicate with a host on the Internet, NAT can be used to convert the local IP to a global IP.

In the past, NAT mapped local IPs and global IPs one-to-one. In this way, a maximum of n hosts in a private network with n global I
P addresses could access the Internet at the same time. In order to make more effective use of global IP addresses, the commonly used NAT translation table also uses the port number of the transport layer, so that hosts within multiple private networks share a global IP address. NAT using port numbers is also called Network Address and Port Translation NAPT.

## Router structure

Routers can be functionally divided into: routing and packet forwarding.

The packet forwarding structure consists of three parts: switching node
structure, a set of input ports and a set of output ports.

## Router packet forwarding process

- Extract the IP address D of the destination host from the header of the datagram to obtain the destination network address N.
- If N is a network address directly connected to this router, direct delivery is performed;
- If there is a route to a specific host with destination address D in the routing table, the datagram is sent to the next-hop router specified in the table;
- If there is a route to network N in the routing table, the datagram is sent to the next-hop router specified in the routing table;
- If there is a default route in the routing table, the datagram is sent to the default router specified in the routing table;
- Report errors in forwarding packets.

## Routing protocol

Routing protocols are adaptive and can adjust adaptively as network traffic and topology changes.

The Internet can be divided into many smaller autonomous systems (AS), and one AS can use a different routing protocol than other ASs.

Routing protocols can be divided into two major categories:

- Routing within autonomous systems: RIP and OSPF
- Routing between autonomous systems: BGP

### 1. Interior Gateway Protocol RIP

RIP is a distance vector-based routing protocol. Distance refers to the number of hops, and the hop number of a directly connected router is 1. The maximum hop count is 15. If it exceeds 15, it means it is unreachable.

RIP only exchanges its own routing table with neighboring routers at fixed intervals. After several exchanges, all routers will eventually know the shortest distance and next-hop router address to any network in this autonomous system.

Distance vector algorithm:

- For RIP messages sent from the adjacent router with address X, first modify all items in the message, change the address in the next hop field to X, and add 1 to all distance fields;
- For each item in the modified RIP packet, perform the following steps:
 - If there is no destination networ

- If the updated routing table from the neighboring router has not been received within 3 minutes, the neighboring router will be marked as unreachable, that is, the distance will be set to 16.

The RIP protocol is simple to implement and has low overhead. However, the maximum distance that RIP can use is 15, which limits the size of the network. And when the network fails, it takes a relatively long time to transmit this message to all routers.

### 2. Interior Gateway Protocol OSPF

Open Shortest Path First OSPF was developed to overcome the shortcomings of RIP.

Open means that OSPF is not controlled by a single manufacturer, but is published publicly; shortest path first means that it uses the shortest path algorithm SPF proposed by Dijkstra.

OSPF has the following characteristics:

- Send information to all routers in this autonomous system. This method is the flooding method.
- The information sent is the link status with adjacent routers. The link status includes which routers are connected to it and the link measurement. The measurement is represented by cost, distance, delay, bandwidth, etc.
- The router will only send information when the link status changes.

All routers have a consistent topology diagram of the entire network. Compared with RIP, the OSPF update process converges quickly.

### 3. Exterior Gateway Protocol BGP

BGP (Border Gateway Protocol, Border Gateway Protocol)

Routing between ASs is difficult mainly due to:

- The Internet is huge;
- Each AS uses different routing protocols internally, and the metric of the path cannot be accurately defined;
- Routing selection between ASs must consider relevant policies. For example, some ASs are not willing to let other ASs pass through.

BGP can only find a better route, not the best route.

Each AS must be configured with a BGP speaker to exchange routing information by establishing a TCP connection between two adjacent BGP speakers.
