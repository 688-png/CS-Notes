## Virtual memory

The purpose of virtual memory is to expand physical memory into larger logical memory, so that the program can obtain more available memory.

In order to better manage memory, the operating system abstracts memory into an address space. Each program has its own address space, which is divided into multiple blocks, each block is called a page. The pages are mapped into physical memory, but do not need to be mapped into contiguous physical memory, nor do all pages need to be in physical memory. When a program references a page that is not in physical memory, the hardware performs the necessary mapping, loads the missing portion into physical memory, and re-executes the failed instruction.

As can be seen from the above description, virtual memory allows the program to not map every page in the address space to physical memory, which means that a program does not need to be loaded into the memory to run, which makes it possible to run large programs with limited memory. For example, if a computer can generate 16-bit addresses, then the address space range of a program is 0\~64K. The computer has only 32KB of physical memory, and virtual memory technology allows the computer to run a 64K program.

## Page replacement algorithm

During the running of the program, if the page to be accessed is not in the memory, a page fault interrupt occurs and the page is transferred into the memory. At this time, if there is no free space in the memory, the system must transfer a page from the memory to the disk swap area to make room.

The page replacement algorithm is similar to the cache elimination strategy. The memory can be regarded as the cache of the disk. In the cache system, the size of the cache is limited. When a new cache arrives, some of the existing caches need to be eliminated so that there is space to store new cache data.

The main goal of the page replacement algorithm is to minimize the page replacement frequency (which can also be said to be the lowest page fault rate).

### 1. Best

> OPT, Optimal replacement algorithm

The selected page to be swapped out will not be accessed for the longest period of time, which usually guarantees the lowest page fault rate.

It is a theoretical algorithm because there is no way to know how long it will take for a page to no longer be visited.

Example: A system allocates three physical blocks to a process, and has the following page reference sequence:

```html
7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2, 1, 2, 0, 1, 7, 0, 1
```

When starting to run, first load the three pages 7, 0, and 1 into the memory. When the process wants to access page 2, a page fault interrupt occurs, and page 7 will be swapped out, because page 7 takes the longest to be accessed again.

### 2. Most recently unused

> LRU, Least Recently Used

Although there is no way to know how a page will be used in the future, you can know how a page has been used in the past. LRU swaps out the most recently unused pages.

In order to implement LRU, a linked list of all pages needs to be maintained in memory. When a page
When the page is accessed, this page is moved to the head of the linked list. This ensures that the page at the end of the linked list is the most recently visited page.

Because each access requires updating the linked list, LRU implemented in this way is expensive.

```html
4, 7, 0, 7, 1, 0, 1, 2, 1, 2, 6
```

### 3. Not used recently

> NRU, Not Recently Used

Each page has two status bits: R and M. The page's R=1 is set when the page is accessed, and M=1 is set when the page is modified. The R bit will be cleared periodically. Pages can be divided into the following four categories:

- R=0, M=0
- R=0, M=1
- R=1, M=0
- R=1, M=1

When a page fault occurs, the NRU algorithm randomly selects a page from the non-empty class with the smallest class number and swaps it out.

NRU preferentially swaps out dirty pages that have been modified (R=0, M=1), rather than frequently used clean pages (R=1, M=0).

### 4. First in, first out

> FIFO, First In First Out

The page selected to be swapped out is the page entered first.

This algorithm will swap out frequently accessed pages, causing the page fault rate to increase.

### 5. Second Chance Algorithm

The FIFO algorithm may replace frequently used pages. To avoid this problem, make a simple modification to the algorithm:

Set the R bit of the page to 1 when the page is accessed (read or write). When replacement is needed, check the R bit of the oldest page. If the R bit is 0, then the page is old and unused and can be replaced immediately; if it is 1, clear the R bit to 0, put the page at the end of the linked list, modify its loading time to make it look like it was just loaded, and then continue searching from the head of the linked list.

## Section page style

The program's address space is divided into multiple segments with independent address spaces, and the address space on each segment is divided into pages of the same size. This not only has the sharing and protection of the segmented system, but also has the virtual memory function of the paging system.

## Comparison between paging and segmentation

- Transparency to programmers: paging is transparent, but segmentation requires the programmer to explicitly divide each segment.

- Dimensions of the address space: Paging is a one-dimensional address space, segmentation is a two-dimensional one.

- Whether the size can be changed: The size of the page is immutable, and the size of the segment can be changed dynamically.

- Reasons for appearance: Paging is mainly used to implement virtual memory to obtain a larger address space; segmentation is mainly used to enable programs and data to be divided into logically independent address spaces and to facilitate sharing and protection.
