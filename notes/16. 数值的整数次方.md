## Title description

Given a floating-point number x of type double and an integer n of type int, find x raised to the power n.

## Problem-solving ideas

<!-- <div align="center"><img src="https://latex.codecogs.com/gif.latex?x^n=\left\{\begin{array}{rcl}x^{n/2}*x^{n/2}&&{n\%2=0}\\x*(x^{n/2}*x^{n/2})&&{n\%2=1}\end{array}\right." class="mathjax-pic"/></div> <br> -->

The most intuitive solution is to repeatedly multiply x n times, x\*x\*x...\*x, then the time complexity is O(N). Because multiplication is commutative, the above operation can be split into two halves (x\*x..\*x)\* (x\*x..\*x). The calculations for both halves are the same, so they only need to be calculated once. And for newly disassembled calculations, you can continue to disassemble them. This is the divide-and-conquer idea, which splits the size of the original problem into multiple smaller sub-problems, and finally merges the solutions to the sub-problems.

The sub-problem in this question is x<sup>n/2</sup>. When merging the sub-problems, just multiply the solutions of the sub-problems by themselves. But if n is not an even number, then split in half there will be one x left, and you need to multiply by one more x when merging the sub-problems.

Because (x\*x)<sup>n/2</sup> can be solved recursively, and n is reduced by half with each recursion, the time complexity of the entire algorithm is O(logN).

```java
public double Power(double x, int n) {
    boolean isNegative = false;
    if (n < 0) {
        n = -n;
        isNegative = true;
    }
    double res = pow(x, n);
    return isNegative ? 1 / res : res;
}

private double pow(double x, int n) {
    if (n == 0) return 1;
    if (n == 1) return x;
    double res = pow(x, n / 2);
    res = res * res;
    if (n % 2 != 0) res *= x;
    return res;
}
```
