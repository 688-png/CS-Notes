## Circuit switching and packet switching

### 1. Circuit switching

Circuit switching is used in telephone communication systems. Before two users can communicate, a dedicated physical link needs to be established and the link is always occupied during the entire communicat
ion process. Since it is impossible to use transmission lines all the time during communication, the utilization rate of circuit switching is very low, often less than 10%.

### 2. Packet switching

Each packet has a header and a tail, including control information such as source address and destination address. Transmitting multiple packets at the same time on the same transmission line will not affect each other. Therefore, multiple packets are allowed to be transmitted at the same time on the same transmission line. That is to say, packet switching does not need to occupy the transmission line.

In a post office communication system, after the post office receives a piece of mail, it first stores it, and then forwards the mail with the same destination to the next destination. This process is the store and forward process. Packet switching also uses the store and forward process.

## Delay

Total delay = queuing delay + processing delay + transmission delay + propagation delay

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/4b2ae78c-e254-44df-9e37-578e2f2bef52.jpg" width="380"/> </div><br>

### 1. Queuing delay

The amount of time a packet waits in the router's input and output queues depends on the current traffic volume on the network.

### 2. Processing delay

The time it takes for a host or router to process a packet when it receives it, such as analyzing the header, extracting data from the packet, performing error checking, or finding the appropriate route.

### 3. Transmission delay

The time it takes for a host or router to transmit a data frame.

## Computer network architecture

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/0fa6c237-a909-4e2a-a771-2c5485cd8ce0.png" width="450"/> </div><br>

### 1. Five-layer protocol

- **Application layer**: Provides data transmission services for specific applications, such as HTTP, DNS and other protocols. The data unit is a message.

- **Transport
layer**: Provides general data transmission services for processes. Since there are many application layer protocols, defining a common transport layer protocol can support an increasing number of application layer protocols. The transport layer includes two protocols: Transmission Control Protocol TCP, which provides connection-oriented and reliable data transmission services, and the data unit is a message segment; User Datagram Protocol UDP, which provides connectionless, best-effort data transmission services, and the data unit is a user datagram. TCP mainly provides integrity services, and UDP mainly provides timeliness services.

- **Network layer**: Provides data transmission services to the host. The transport layer protocol provides data transmission services for processes in the host. The network layer encapsulates the message segments or user datagrams passed down by the transport layer into groups.

- **Data Link Layer**: The network layer is still aimed at data transmission services between hosts, and there can be many links between hosts. The link layer protocol provides data transmission services for hosts on the same link. The data link layer encapsulates the packets transmitted from the network layer into frames.

- **Physical layer**: Considers how to transmit the data bit stream on the transmission medium, rather than the specific transmission medium. The role of the physical layer is to shield the differences in transmission media and communication means as much as possible so that the data link layer cannot feel these differences.

### 2. OSI

The presentation layer and session layer are used as follows:

- **Presentation layer**: Data compression, encryption and data description, which allows applications to not have to worry about the different internal formats of data in each host.

- **Session Layer**: Establish and manage sessions.

Five-layer protocols do not have a presentation layer and a session layer, leaving these functions to the application developer.

### 3.TCP/IP

It has only four layers, which is equivalent to merging the data link layer and physical layer into the network interface layer in the five-layer protocol.

The TCP/IP architecture does not strictly follow the OSI layering concept, and the application layer may directly use the IP layer or the network interface layer.

### 4. Data transfer process between layers

In the downward process, the header or tail required by the lower layer protocol needs to be added, and in the upward process, the header and tail are continuously separated.

The router only has the following three layers of protocols. Because the router is located in the core of the network and does not need to provide services for processes or applications, there is no need for the transport layer and application layer.
