## Domain name system

DNS is a distributed database that provides services for converting host names and IP addresses to and from each other. The distributed database here means that each site only retains its own part of the data.

Domain names have a hierarchical structure, from top to bottom: root domain name, top-level domain name, and second-level domain name.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/b54eeb16-0b0e-484c-be62-306f57c40d77.jpg"/> </div><br>

DNS can be transmitted using UDP or TCP, and the port number used is 53. In most cases, DNS uses UDP for transmission, which requires both the domain name resolver and the domain name server to handle timeouts and retransmissions themselves to ensure reliability. TCP is used for transmission in two cases:

- If the returned response exceeds 512 bytes (UDP only supports a maximum of 512 bytes of data).
- Zone transfer (zone transfer is the transfer of the changed data from the primary domain name server to the secondary domain name server).

## File Transfer Protocol

FTP uses TCP to connect, which requires two connections to transfer a file:

- Control connection: The server opens port number 21 and waits for the client's connection. After the client actively establishes a connection, it uses this connection to transmit the client's command to the server and returns the server's response.
- Data connection: used to transfer a file data.

Depending on whether the data connection is actively established by the server, FTP has two modes: active and passive:

- Active mode: The server actively establishes a data connection. The server's port number is 20, and the client's port number is random, but must be greater than 1024, because 0\~1023 is a well-known port number.

- Passive mode: The client actively establishes a data connection, in which the client's port number is specified by the client itself, and the se

Active mode requires the client to open the port number to the server, and the client's firewall needs to be configured. Passive mode only requires an open port number on the server side and does not require a firewall configuration on the client side. But passive mode can lead to weakened security on the server side because too many port numbers are opened.

## Dynamic Host Configuration Protocol

DHCP (Dynamic Host Configuration Protocol) provides a plug-and-play networking method, and users no longer need to manually configure IP addresses and other information.

The content of DHCP configuration is not only the IP address, but also the subnet mask and gateway IP address.

The working process of DHCP is as follows:

1. The client sends a Discover message. The destination address of the message is 255.255.255.255:67 and the source address is 0.0.0.0:68. It is put into UDP and the message is broadcast to all hosts on the same subnet. If the client and DHCP server are not on the same subnet, a relay agent is required.
2. After receiving the Discover message, the DHCP server sends an Offer message to the client. This message contains the information required by the client. Because the client may receive information from multiple DHCP servers, the client needs to choose.
3. If the client selects the information provided by a DHCP server, it sends a Request message to the DHCP server.
4. The DHCP server sends an Ack message, indicating that the client can use the information provided to it at this time.

## Remote login protocol

TELNET is used to log in to the remote host and output from the remote host is returned.

TELNET can accommodate many computer and operating system differences, such as newline character definitions for different operating system systems.

## Email Protocol

An email system consists of three parts: user agent, mail server, and mail protocol.

Email protocols include sending protocols and reading protocols. SMTP is commonly used as sending protocol, and POP3 and IMAP are commonly used as reading protocols.

### 1. SMTP

SMTP can only send ASCII, while Internet Mail Extensions MIME can send binary files. MIME does not change or replace SMTP, but increases the structure of the email body and defines non-ASCII encoding rules.

## Web page request process

### 1. DHCP configuration host information

- Assuming that the host does not have an IP address and other information at the beginning, it needs to use DHCP to obtain it first.

- The host generates a DHCP request message and puts this message into a UDP segment with destination port 67 and source port 68.

- The segment is placed in an IP datagram with a broadcast IP destination address (255.255.255.255) and a source IP address (0.0.0.0).

- The datagram is placed in a MAC frame with the destination address FF:\<zero-width space\>FF:\<zero-width space\>FF:\<zero-width space\>FF:\<zero-width space\>FF:FF, which is broadcast to all devices connected to the switch.

- After the DHCP server connected to the switch receives the broadcast frame, it continuously decomposes it upwards to obtain IP datagrams, UDP segments, and DHCP request messages, and then generates a DHCP ACK message. This message contains the following information: IP address, IP address of the DNS server, IP address of the default gateway router, and subnet mask. The message is put into a UDP segment, which is put into an IP datagram, and finally into a MAC frame.

- The destination address of this frame is the MAC address of the requesting host. Because the switch has self-learning capability, after the host sent the broadcast frame before, it recorded the switching table entry of the MAC address to its forwarding interface, so now the switch can directly know to which interface the frame should be sent.

- After receiving the frame, the host continuously decomposes it to obtain DHCP messages. It then configures its IP address, subnet mask, and DNS server IP address, and installs a default gateway in its IP forwarding table.

### 2. ARP resolves MAC address

- The host generates a TCP socket through the browser, and the socket sends an HTTP request to the HTTP server. In order to generate
this socket, the host needs to know the IP address corresponding to the website's domain name.

- The host generates a DNS query message with port number 53 because the port number of the DNS server is 53.

- The DNS query message is put into an IP datagram whose destination address is the IP address of the DNS server.

- The IP datagram is put into an Ethernet frame, which is sent to the gateway router.

- The DHCP process only knows the IP address of the gateway router. In order to obtain the MAC address of the gateway router, the ARP protocol is required.

- The host generates an ARP query message containing the IP address of the gateway router, and puts the ARP query message into a broadcast destination address (FF:\<zero-width space\>FF:\<zero-width space\>FF:\<zero-width space\>FF:\<zero-width space\>FF:FF) and sends the Ethernet frame to the switch, and the switch forwards the frame to all connected devices, including the gateway router.

- After the gateway router receives the frame, it continuously decomposes the ARP message upwards and finds that the IP address in it matches the IP address of its interface, so it sends an ARP reply message, including its MAC address, back to the host.

### 3. DNS resolves domain name

- Once you know the MAC address of your gateway router, you can continue the DNS resolution process.

- After receiving the Ethernet frame containing the DNS query message, the gateway router extracts the IP datagram and determines the router to which the IP datagram should be forwarded based on the forwarding table.

- Because the router has two routing protocols: interior gateway protocol (RIP, OSPF) and exterior gateway protocol (BGP), the routing table entry for the gateway router to reach the DNS server has been configured in the routing table.

- After arriving at the DNS server, the DNS server extracts the DNS query message and searches the DNS database for the domain name to be resolved.

- After finding the DNS record, send a DNS reply message, put the reply message into a UDP segment, and then put it into an IP datagram, forward it back to the gateway router through the router, and reach the host through the Ethernet switch.

### 4. HTTP request page

- With the IP address of the HTTP server, the host can generate a TCP socket, which will be used to send HTTP GET messages to the web server.

- Before generating a TCP socket, a three-way handshake with the HTTP server must be performed to establish the connection. Generate a TCP SYN segment with destination port 80 and send the segment to the HTTP server.

- After receiving the segment, the HTTP server generates a TCP SYN ACK segment and sends it back to the host.

- After the connection is established, the browser generates an HTTP GET message and delivers it to the HTTP server.

- The HTTP server reads the HTTP GET message from the TCP socket, generates an HTTP response message, puts the Web page content into the message body, and sends it back to the host.

-
After receiving the HTTP response message, the browser extracts the content of the Web page, and then renders and displays the Web page.
