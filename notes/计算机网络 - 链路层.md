## Basic questions

### 1. Encapsulation into frame

Add headers and trailers to the packets passed from the network layer to mark the beginning and end of the frame.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/29a14735-e154-4f60-9a04-c9628e5d09f4.png" width="300"/> </div><br>

### 2. Transparent transmission

Transparency means that an actual thing appears as if it does not exist.

Frames are delimited by headers and trailers. If the data part of the frame contains the same content as the header and trailer, the start and end positions of the frame will be incorrectly determined. It is necessary to insert escape characters before the content with the same beginning and end in the data part. If an escape character appears in the data part, add an escape character before the escape character. The original data can be restored after processing at the receiving end. The content transmitted transparently in this process is escape characters, and users are not aware of the existence of escape characters.

### 3. Error detection

Cyclic redundancy check (CRC) is currently widely used in the data link layer to check for bit errors.

## Channel classification

### 1. Broadcast channel

One-to-many communication, data sent by one node can be received by all nodes on the broadcast channel.

All nodes send data on the same broadcast channel, so special control methods are required for coordination to avoid conflicts (conflicts are also called collisions).

There are two main control methods for coordination, one is to use channel multiplexin
g technology, and the other is to use CSMA/CD protocol.

### 2. Point-to-point channel

One-to-one communication.

Because there will be no collision, it is relatively simple and is controlled using the PPP protocol.

## Channel multiplexing technology

### 1. Frequency division multiplexing

All hosts in frequency division multiplexing occupy different frequency bandwidth resources at the same time.

### 2. Time division multiplexing

All hosts in time division multiplexing occupy the same frequency bandwidth resources at different times.

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/67582ade-d44a-46a6-8757-3c1296cc1ef9.png" width="350"/> </div><br>

Frequency division multiplexing and time division multiplexing are used for communication. During the communication process, the host will always occupy part of the channel resources. However, due to the bursty nature of computer data, there is no need for the communication process to occupy channel resources all the time without letting them be used by other users. Therefore, the utilization rate of the channel is not high in these two methods.

### 3. Statistical time division multiplexing

It is an improvement on time division multiplexing. The position of each user in the time division multiplexing frame is not fixed. As long as there is data, it is gathered together to form a statistical time division multiplexing frame and then sent.

Optical frequency division multiplexing. Since the frequency of light is very high, it is customary to express the optical carrier used in terms of wavelength rather than frequency.

## CSMA/CD Protocol

CSMA/CD stands for Carrier Sense Multipoint Access/Collision Detection.

- **Multi-point access**: Indicates that this is a bus-type network, and many hosts are connected to the bus in a multi-point manner.
- **Carrier Sense**: Each host must constantly monitor the channel. Before sending, if the channel is detected to be in use, it must wait.
- **Collision Detection**: During transmission, if another host is monitoring the channel and is sending data, it means a collision has occurred. Although each host has monitored that the channel is idle before sending data, collisions may still occur due to the propagation delay of electromagnetic waves.

Record the end-to-end propagation delay as τ. The first station to send can know whether a collision has occurred after at most 2τ. We call 2τ the **contention period**. Only when no collision is detected after the contention period can we be sure that this transmission will not cause a collision.

When a collision occurs, the station should stop sending and wait for a period of time before sending again. This time is determined using the **Truncated Binary Exponential Backoff Algorithm**. Randomly pick a number from the discrete integer set {0, 1, .., (2<sup>k</sup>-1)}, denoted as r, and then take r times the contention period as the retransmission waiting time.

## MAC address

The MAC address is a link layer address, 6 bytes (48 bits) long, that uniquely identifies a network adapter (network card).

A host has as many MAC addresses as it has network adapters. For example, laptops commonly have wireless network adapters and wired network adapters, so they have two MAC addresses.

## Ethernet

Ethernet is a star topology local area network.

In the early days, hubs were used for connection. The hub is a physical layer device that operates on bits rather than frames. When a bit arrives at the interface, the hub regenerates the bit and amplifies its energy intensity, thereby expanding the transmission distance of the network, and then sends the bit to all other interfaces. If a hub receives frames from two different interfaces at the same time, a collision occurs.

Currently, Ethernet uses switches to replace hubs. The switch is a link layer device that does not collide and can store and forward based on the MAC address.

Ethernet frame format:

- **Type**: Mark the protocol used by the upper layer;
- **Data**: The length is between 46-1500, if it is too small, it needs to be padded;
- **FCS**: Frame check sequence, using the CRC check method;

## Switch

The switch has self-learning capability and learns the contents of the switching table, which stores the mapping of MAC addresses to interfaces.

Because of this self-learning capability, the switch is a plug-and-play device that does not require the network administrator to manually configure the switching table contents.

In the figure below, the switch has 4 interfaces. When host A sends a data frame to host B, the switch writes the mapping from host A to interface 1 into the switching
table. In order to send a data frame to B, first check the switching table. At this time, there is no entry for host B. Then host A sends a broadcast frame. Host C and host D will discard the frame. When host B responds to the frame and sends a data packet to host A, the switch looks up the switching table and finds that the interface mapped by host A is 1, so it sends the data frame to interface 1. At the same time, the switch adds the mapping of host B to interface 2.

## Virtual LAN

A virtual LAN can establish logical groups that are independent of physical location. Only members in the same virtual LAN will receive link layer broadcast information.

For example, in the figure below (A1, A2, A3, A4) belongs to a virtual LAN, the broadcast sent by A1 will be received by A2, A3, and A4, but other stations will not receive it.

Virtual LANs are built using VLAN trunking, where a special interface on each switch is set up as a trunk interface to interconnect VLAN switches. IEEE defines an extended Ethernet frame format 802.1Q, which adds a 4-byte header VLAN tag to the standard Ethernet frame to indicate which virtual LAN the frame belongs to.
