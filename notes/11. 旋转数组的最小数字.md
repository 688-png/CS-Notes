## Title description

Moving the first few elements of an array to the end of the array is called rotation of the array. Inputs a rotation of a non-decreasingly sorted array and outputs the smallest element of the rotated array.

## Problem-solving ideas

Splitting the rotated array in half results in a new rotated array containing the smallest element, and a non-decreasingly sorted array. The length of the new rotated array is half of the original array, thus reducing the size of the problem by half. The time complexity of this half-property algorithm is O(log<sub>2</sub>N).

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/424f34ab-a9fd-49a6-9969-d76b42251365.png" width="300px"> </div><br>

The key to the problem at this time is to determine which of the two arrays obtained by dividing in half is a rotating array and which one is a non-decreasing array. We can easily know that the first element of a non-decreasing array must be less than or equal to the last element.

Solve by modifying the binary search algorithm (l represents low, m represents mid, h represents high):

- When nums[m] \<= nums[h], it means that the array in the [m, h] interval is a non-decreasing array, and the array in the [l, m] interval is a rotating array. In this case, let h = m;
- Otherwise the array in the interval [m + 1, h] is a rotated array, let l = m + 1.

```java
public int minNumberInRotateArray(int[] nums) {
    if (nums. length == 0)
        return 0;
    int l = 0, h = nums.length - 1;
    while (l < h) {
        int m = l + (h - l) / 2;
        if (nums[m] <= nums[h])
            h = m;
        else
            l = m + 1;
    }
    return nums[l];
}
```

If the array elements are allowed to be repeated, a special situation will occur: nums[l] == nums[m] == nums[h]. At this time, it is impossible to determine which interval the solution is in, and you need to switch to sequential search. For example, for the array {1,1,1,0,1}, the numbers pointed to by l, m, and h are all 1. At this time, it is impossible to know in which interval the smallest number 0 is.

```java
public int minNumberInRotateArray(int[] nums) {
    if (nums. length == 0)
        return 0;
    int l = 0, h = nums.length - 1;
    while (l < h) {
        int m = l + (h - l) / 2;
        if (nums[l] == nums[m] && nums[m] == nums[h])
            return minNumber(nums, l, h);
        else if (nums[m] <= nums[h])
            h = m;
        else
            l = m + 1;
    }
    return nums[l];
}
```
