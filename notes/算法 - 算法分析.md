## Mathematical model

### 1. Approximation

N<sup>3</sup>/6-N<sup>2</sup>/2+N/3 \~ N<sup>3</sup>/6. Use \~f(N) to denote all functions whose result divided by f(N) approaches 1 as N increases.

### 2. Order of magnitude growth

The growth of N<sup>3</sup>/6-N<sup>2</sup>/2+N/3 is of the order of O(N<sup>3</sup>). The order of magnitude growth isolates the algorithm from its specific implementation. The growth magnitude of an algorithm is O(N<sup>3</sup>) regardless of whether it is implemented in Java and whether it runs on a specific computer.

### 3. Inner loop

The most frequently executed instructions determine the total execution time of the program. These instructions are called the inner loop of the program.

### 4. Cost model

Evaluate algorithms using a cost model, such as the number of array accesses.

## Notes

### 1. Large constant

When approximating, if the constant coefficient of the low-level term is large, the approximate result will be wrong.

### 2. Caching

Computer systems use caching technology to organize memory, and accessing adjacent elements in an array is much faster than accessing non-adjacent elements.

### 3. Guarantee of worst-case performance

For software in nuclear reactors, pacemakers or brake controllers, worst-case performance is critical.

### 4. Randomization algorithm

By shuffling the input, the algorithm's dependence on the input is removed.

### 5. Equally allocated analysis

Divide the total cost of all operations by the total number of operations to spread the cost evenly. For example, the number of array accesses required for N consecutive push() calls on an empty stack is N+4+8+16+...+2N=5N-4 (N is the number of times to write elements to the array, and the rest are the number of array accesses required for copying when adjusting the array size). The average number of array accesses after amortization is a constant.

## ThreeSum

ThreeSum is used to count the number of triples in an array whose sum is 0.

```java
public interface ThreeSum {
    int count(int[] nums);
}
```

### 1. ThreeSumSlow
The inner loop of this algorithm is the `if (nums[i] + nums[j] + nums[k] == 0)` statement. The total number of executions is N(N-1)(N-2) = N<sup>3</sup>/6-N<sup>2</sup>/2+N/3. Therefore, its approximate number of executions is \~N<sup>3</sup>/6, and the order of magnitude increase is O(N<sup>3</sup>).

```java
public class ThreeSumSlow implements ThreeSum {
    @Override
    public int count(int[] nums) {
        int N = nums.length;
        int cnt = 0;
        for (int i = 0; i < N; i++) {
            for (int j = i + 1; j < N; j++) {
                for (int k = j + 1; k < N; k++) {
                    if (nums[i] + nums[j] + nums[k] == 0) {
                        cnt++;
                    }
                }
            }
        }
        return cnt;
    }
}
```

### 2. ThreeSumBinarySearch

Sort the array, sum the two elements, and use the binary search method to find whether the opposite number of the sum exists. If it exists, it means that there is a triplet whose sum is 0.

It should be noted that this solution can only be used if the arrays do not contain identical elements, otherwise the result of the binary search will be wrong.

This method can reduce the growth of ThreeSum algorithm to O(N<sup>2</sup>logN).

```java
public class ThreeSumBinarySearch implements ThreeSum {

    @Override
    public int count(int[] nums) {
        Arrays.sort(nums);
        int N = nums.length;
        int cnt = 0;
        for (int i = 0; i < N; i++) {
            for (int j = i + 1; j < N; j++) {
                int target = -nums[i] - nums[j];
                int index = BinarySearch.search(nums, target);
                // It should be noted that the subscript here must be greater than j, otherwise statistics will be repeated.
                if (index > j) {
                    cnt++;
                }
            }
        }
        return cnt;
    }
}
```

```java
public class BinarySearch {

    public static int search(int[] nums, int target) {
        int l = 0, h = nums.length - 1;
        while (
l <= h) {
            int m = l + (h - l) / 2;
            if (target == nums[m]) {
                return m;
            } else if (target > nums[m]) {
                l = m + 1;
            } else {
                h = m - 1;
            }
        }
        return -1;
    }
}
```

### 3. ThreeSumTwoPointer

A more efficient method is to sort the array first and then use double pointers to search, with a time complexity of O(N<sup>2</sup>).

The same does not apply to the case where there are duplicate elements in the array.

```java
public class ThreeSumTwoPointer implements ThreeSum {

    @Override
    public int count(int[] nums) {
        int N = nums.length;
        int cnt = 0;
        Arrays.sort(nums);
        for (int i = 0; i < N - 2; i++) {
            int l = i + 1, h = N - 1, target = -nums[i];
            while (l < h) {
                int sum = nums[l] + nums[h];
                if (sum == target) {
                    cnt++;
l++;
                    h--;
                } else if (sum < target) {
                    l++;
                } else {
                    h--;
                }
            }
        }
        return cnt;
    }
}
```

## Magnification experiment

If T(N) \~ aN<sup>b</sup>logN, then T(2N)/T(N) \~ 2<sup>b</sup>.

For example, for the brute-force ThreeSum algorithm, the approximate time is \~N<sup>3</sup>/6. Conduct the following experiment: run the algorithm multiple times, and the N value taken each time is twice that of the previous time. The time of each execution is counted, and the ratio of this running time to the previous running time is calculated. The following results are obtained:

| N | Time(ms) | Ratio |
| :---: | :---: | :---: |
| 500 | 48 | / |
| 1000 | 320 | 6.7 |
| 2000 | 555 | 1.7 |
| 4000 | 4105 | 7.4 |
| 8000 | 33575 | 8.2 |
| 16000 | 268909 | 8.0 |

It can be seen that T(2N)/T(N) \~ 2<sup>3</sup>, so T(N) \~ aN<sup>3</sup>logN can be determined.

```java
public class RatioTest {

    public static void main(String[] args) {
        int N = 500;
        int loopTimes = 7;
        double preTime = -1;
        while (loopTimes-- > 0) {
            int[] nums = new int[N];
            StopWatch.start();
            ThreeSum threeSum = new ThreeSumSlow();
            int cnt = threeSum.count(nums);
            System.out.println(cnt);
            double elapsedTime = StopWatch.elapsedTime();
            double ratio = preTime == -1 ? 0 : elapsedTime / preTime;
            System.out.println(N + " " + elapsedTime + " " + ratio);
            preTime = elapsedTime;
            N *= 2;
        }
    }
}
```

```java
public class StopWatch {

    private static long start;


    public static void start() {
        start = System.currentTimeMillis();
    }


    public static double elapsedTime() {
        long now = System.currentTimeMillis();
        return (now - start) / 1000.0;
    }
}
```
