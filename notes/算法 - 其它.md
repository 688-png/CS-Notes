This is a classic recursive problem, which can be solved in three steps:

① Move n-1 disks from from -\> buffer

If there is only one disk, then only one move operation is required.

From the above discussion, we can know that a<sub>n</sub> = 2 * a<sub>n-1</sub> + 1, obviously a<sub>n</sub> = 2<sup>n</sup> - 1, n disks need to move 2<sup>n</sup> - 1 times.

```java
public class Hanoi {
    public static void move(int n, String from, String buffer, String to) {
        if (n == 1) {
            System.out.println("from " + from + " to " + to);
            return;
        }
        move(n - 1, from, to, buffer);
        move(1, from, buffer, to);
        move(n - 1, buffer, from, to);
    }

    public static void main(String[] args) {
        Hanoi.move(3, "H1", "H2", "H3");
    }
}
```

## Huffman coding

The raw data is compressed by encoding it based on its frequency of occurrence.

For example, for a text file, the number of occurrences of various characters is as follows:

- a : 10
- b : 20
- c : 40
- d : 80

Each character can be converted to a binary encoding, such as a to 00, b to 01, c to 10, and d to 11. This is the simplest encoding method, which does not consider the weight (frequency of occurrence) of each character. Huffman coding uses a greedy strategy to make the encoding of the most frequent characters the shortest, thereby ensuring the shortest overall encoding length.

First, a Huffman tree is generated. In each generation process, the two nodes with the least frequency are selected, and a new node is generated as their parent node, and the frequency of the new node is the sum of the two nodes. The reason for selecting the least frequent is that the generation process makes the node selected first be located at a lower level of the tree, so the required coding length is longer, and less frequency can make the total coding length smaller.

When generating the encoding, start from the root node, traverse to the left and add binary bits 0, and add bina
ry bits 1 to the right until you traverse to the leaf node. The encoding of the character represented by the leaf node is this path encoding.

```java
public class Huffman {

    private class Node implements Comparable<Node> {
        char ch;
        int freq;
        boolean isLeaf;
        Node left, right;

        public Node(char ch, int freq) {
            this.ch = ch;
            this.freq = freq;
            isLeaf = true;
        }

        public Node(Node left, Node right, int freq) {
            this.left = left;
            this.right = right;
            this.freq = freq;
            isLeaf = false;
        }

        @Override
        public int compareTo(Node o) {
            return this.freq - o.freq;
        }
    }

    public Map<Character, String> encode(Map<Character, Integer> frequencyForChar) {
        PriorityQueue<Node> priorityQueue = new PriorityQueue<>();
        for (Character c : frequencyForChar.keySet()) {
            priorityQueue.add(new Node(c, frequencyForChar.get(c)));
        }
        whi
le (priorityQueue.size() != 1) {
            Node node1 = priorityQueue.poll();
            Node node2 = priorityQueue.poll();
            priorityQueue.add(new Node(node1, node2, node1.freq + node2.freq));
        }
        return encode(priorityQueue.poll());
    }

    private Map<Character, String> encode(Node root) {
        Map<Character, String> encodingForChar = new HashMap<>();
        encode(root, "", encodingForChar);
        return encodingForChar;
    }

    private void encode(Node node, String encoding, Map<Character, String> encodingForChar) {
        if (node.isLeaf) {
            encodingForChar.put(node.ch, encoding);
            return;
        }
        encode(node.left, encoding + '0', encodingForChar);
        encode(node.right, encoding + '1', encodingForChar);
    }
}
```
