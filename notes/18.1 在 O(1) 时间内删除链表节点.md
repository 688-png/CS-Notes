# 18.1 Delete linked list nodes in O(1) time

## Problem-solving ideas

① If the node is not the tail node, you can directly assign the value of the next node to the node, then point the node to the next node, and then delete the next node. The time complexity is O(1).

② Otherwise, you need to traverse the linked list first, find the previous node of the node, and then let the previous node point to null. The time complexity is O(N).

<div align="center"> <img src="https://cs-notes-1256109796.cos.ap-guangzhou.myqcloud.com/4bf8d0ba-36f0-459e-83a0-f15278a5a157.png" width="600"/> </div><br>

To sum up, if N operations are performed, the approximate number of times the node needs to be operated is N-1+N=2N-1, where N-1 represents the total number of times that each of the N-1 nodes that is not the tail node operates the node with a time complexity of O(1), and N represents the total number of times that one tail node operates the node with a time complexity of O(N). (2N-1)/N \~ 2, so the average time complexity of this algorithm is O(1).

```java
public ListNode deleteNode(ListNode head, ListNode tobeDelete) {
    if (head == null || tobeDelete == null)
        return null;
    if (tobeDelete.next != null) {
        //The node to be deleted is not the tail node
        ListNode next = tobeDelete.next;
        tobeDelete.val = next.val;
        tobeDelete.next = next.next;
    } else {
        if (head == tobeDelete)
             // only one node
            head = null;
        else {
            ListNode cur = head;
            while (cur.next != tobeDelete)
                cur = cur.next;
            cur.next = null;
        }
    }
    return head;
}
```
